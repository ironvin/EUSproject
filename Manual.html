Energy Usage System Manual

Running the Program

When you log in as the Manager, the system loads all existing buildings, units, and usage records, then presents you with a numbered menu. You interact with the app entirely through these menu choices. Each option provides different types of information or performs an action.

Main Menu Options:

  1. Exit the program
  2. View building overview
  3. View monthly summary for all buildings
  4. View per-unit usage and thresholds
  5. View alerts and export to CSV
  6. Demo to simulate an over-usage scenario
  7. Export building and unit data to CSV
  8. Export usage history for charting
  9. Show a usage bar chart inside the console

Note: Running the Program as the Administrator will prevent you from using Options 5 and 6.

Option 1: Building Overview

Shows an overview of every building in the system. For each building, you see the energy rate it uses and the number of units it contains. You get a simple numbered list of every building, which is useful for a quick top-level snapshot.

Option 2: Monthly Summary

Shows a monthly summary for all buildings at once. It displays total energy consumption and total cost for the month for each building. This gives you a bird's-eye view of which buildings consume the most energy and how much they cost.

Option 3: Per-Unit Usage

Shows detailed per-unit usage for the month. It groups units under their respective buildings. Each unit displays its energy usage, its threshold, and whether the usage is within acceptable limits or exceeds the threshold. This is where the Manager checks whether individual tenants or rooms are hitting concerning levels.

Option 4: Alerts

Shows any alerts for the month. Alerts appear only when a unit goes past its threshold. If anything exceeds its limit, it will appear here. This option also lets you create a CSV file containing the alerts, allowing you to review them or share them externally.

Option 5: Demo Mode

A demo tool that artificially simulates an over-usage event in one unit. After running it, you can go back to the alerts section to see a new alert appear. This option is mainly for testing or demonstration.

This option can only be used from a Manager account.

Option 6: Export Building Data

Exports all building and unit data into CSV files. This includes building names, rates, unit IDs, thresholds, and any other identifying information the system manages. Managers can open these files in external tools for auditing or reporting.

This option can only be used from a Manager account.

Option 7: Export Usage History

Exports usage over time into a CSV tailored for charting. This is meant to be opened in spreadsheet software so you can easily generate line graphs or trend visuals.

Option 8: Console Chart

Prints a simple bar-style chart directly in the console. Each building is displayed with a bar scaled to its total usage for the month. This gives a quick visual comparison without leaving the app.

Option 0: Exit

Exits the Manager interface entirely.
